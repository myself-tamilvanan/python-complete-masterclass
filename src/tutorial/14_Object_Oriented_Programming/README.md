# Chapter 14: Object-Oriented Programming (OOP)

## Overview
Object-Oriented Programming organizes code around objects that combine data (attributes) and behavior (methods). OOP makes code more modular, reusable, and easier to understand.

## Topics Covered
- Classes and objects
- __init__ constructor method
- Instance attributes and methods
- Inheritance
- Method overriding
- Object lifecycle (__init__ and __del__)
- Magic/dunder methods

## Key Concepts
- **Class**: A blueprint for creating objects
- **Object/Instance**: A specific realization of a class
- **Attribute**: Data stored in an object
- **Method**: A function defined inside a class
- **self**: Reference to the current object instance
- **Constructor (__init__)**: Called when creating an object
- **Inheritance**: A class can inherit from another class
- **Override**: Redefine a parent method in child class

## OOP Principles
| Principle     | Description                                        |
|---------------|----------------------------------------------------|
| Encapsulation | Bundle data and methods together in a class        |
| Inheritance   | Create new classes based on existing ones          |
| Polymorphism  | Different objects respond to same method name      |
| Abstraction   | Hide complex details behind simple interfaces      |
