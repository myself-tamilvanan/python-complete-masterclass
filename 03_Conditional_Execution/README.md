# Chapter 3: Conditional Execution

## Overview
Conditional execution allows programs to make decisions. This is what makes programs "intelligent" — they can do different things based on different situations.

## Topics Covered
- if statements
- else and elif clauses
- Comparison operators (==, !=, <, >, <=, >=)
- Logical operators (and, or, not)
- Nested conditionals
- try/except for error handling
- Guardian patterns

## Key Concepts
- **Condition**: An expression that evaluates to True or False
- **Boolean**: A value that is either True or False
- **Indentation**: Python uses 4 spaces to define code blocks
- **try/except**: A way to handle errors gracefully
- **Guardian Pattern**: Checking conditions before risky operations

## Comparison Operators
| Operator | Meaning               |
|----------|-----------------------|
| ==       | Equal to              |
| !=       | Not equal to          |
| <        | Less than             |
| >        | Greater than          |
| <=       | Less than or equal to |
| >=       | Greater than or equal |

## Logical Operators
| Operator | Description                          |
|----------|--------------------------------------|
| and      | True if BOTH conditions are True     |
| or       | True if AT LEAST ONE is True         |
| not      | Reverses the boolean value           |

## Learning Outcomes
- Write if/elif/else decision structures
- Use comparison and logical operators correctly
- Handle runtime errors with try/except
- Write defensive code using guard patterns
